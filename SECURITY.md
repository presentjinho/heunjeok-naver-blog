# 보안 원칙

## 서버 기초

- `npm start`는 허용 목록의 공개 파일만 제공합니다. 서버 코드와 환경 파일은 내려주지 않습니다.
- API는 허용한 Origin의 JSON 요청만 받고, 요청 본문을 응답이나 감사 로그에 남기지 않습니다.
- 요청 본문은 4KB, 기본 요청 속도는 IP별 분당 30회로 제한합니다.
- OpenAI·네이버 비밀키는 브라우저에 전달하지 않습니다. 실제 값은 배포 서비스의 Secret 저장소에만 둡니다.
- 운영 주소에 사용자 정보·경로·쿼리가 섞이거나 예시 비밀키가 남으면 서버 시작을 거부합니다.
- 익명 세션은 추측하기 어려운 256비트 식별자와 별도 CSRF 토큰을 사용하며 30분 뒤 만료됩니다.
- 세션·OAuth state·IP 속도제한 메모리는 만료 항목을 청소하고 최대 항목 수를 제한해 무한 메모리 증가를 막습니다.
- 세션 쿠키는 `HttpOnly`, `SameSite=Strict`, `Path=/`를 사용하고 운영 HTTPS에서는 `Secure`를 강제합니다.
- 세션 재발급은 이전 세션을 폐기하며, 로그아웃도 동일 출처와 CSRF 검증을 통과해야 합니다.
- 운영 환경의 `SESSION_SECRET`은 32자 미만이면 서버가 시작되지 않습니다. 이후 분산 세션 저장소의 암호화·키 순환에 사용합니다.
- 감사 로그는 사건 종류·요청 ID·성공 여부만 허용 목록으로 남깁니다. 사용자 입력, IP, 쿠키, OAuth state, 토큰, 비밀키는 버립니다.
- 네이버 OAuth `state`는 세션에 묶인 256비트 일회용 값이며 원문 대신 해시만 5분간 보관합니다. 다른 세션·만료·재사용은 거부합니다.
- 인증 코드는 서버에서 한 번만 교환합니다. 액세스 토큰은 네이버 최소 식별값 조회에만 쓰고 세션·브라우저·감사 로그에 저장하지 않습니다.
- 로그인 성공 시 기존 세션을 폐기하고 새 세션으로 회전합니다. 네이버 식별값은 서버 비밀키 기반 HMAC으로 가명 처리합니다.
- AI API는 네이버 로그인 세션과 CSRF를 모두 요구하고 세션별 분당 5회로 제한합니다.
- AI 입력은 허용 글 종류·길이·근거 키를 검사하며 사용자 텍스트를 명령이 아닌 데이터 구역으로 전달합니다.
- AI 출력은 JSON Schema와 서버 검증을 모두 통과해야 합니다. 입력에 없던 근거 ID를 인용한 문단은 사용자에게 전달하지 않습니다.
- OpenAI API 키는 서버 Authorization 헤더에만 사용하며 요청 본문·브라우저·로그에 넣지 않습니다.
- DataLab Client Secret은 서버 헤더에만 사용하고 브라우저에 전달하지 않습니다. 검색어는 60자로 제한하고 제어문자·태그 문자를 제거합니다.
- DataLab 결과는 1시간 캐시하고 로그인 세션별 분당 10회로 제한합니다. 장애 시 임의 추정값을 만들지 않습니다.
- Search Ads 자격 증명은 DataLab·OAuth 자격 증명과 별도 변수로 격리합니다. 키워드 결과는 6시간 캐시하며 약관 확인 기능 플래그가 없으면 외부 호출 자체를 차단합니다.
- 취재 사진 원본은 서버로 보내지 않습니다. 허용 형식·15MB·10장 제한을 적용하고 캔버스 재인코딩으로 메타데이터를 제거한 축소 사본만 IndexedDB에 보관합니다.
- 사진 미리보기에 필요한 `blob:` URL만 CSP 이미지 범위에 추가하며 매 렌더링 때 이전 URL을 폐기합니다.

사용자용 공개판은 정적 GitHub Pages입니다. 경험 입력·말투 카드·초안은 선택한 브라우저에만 저장되며 서버 AI·로그인·자동 발행을 호출하지 않습니다. 저장소의 Node 서버 코드는 개발·검증용 선택 모듈이며 Pages 배포 산출물에 포함되지 않습니다. 사진 원본과 사진 보관함도 서버로 전송하지 않습니다.

## 저장하는 데이터

- 경험 답변, 추가 메모, 사진 설명
- 원문을 제외한 말투 참고 카드
- 생성·편집한 초안

저장하지 않는 데이터:

- 네이버 비밀번호, 로그인 쿠키, 인증 토큰
- OpenAI·네이버 API 키
- 기존 글 원문
- 댓글 원문

## 브라우저 방어

- 기본 Content Security Policy로 스크립트·연결·프레임 범위를 제한한다.
- Referrer Policy는 `no-referrer`다.
- 외부 링크에는 `noopener noreferrer`를 사용한다.
- 사용자 입력은 HTML로 삽입하지 않고 `textContent` 또는 폼 값으로만 처리한다.

## 운영 배포 필수 조건

- 비밀키는 서버 측 secret manager 또는 환경변수에만 저장한다.
- 인증 쿠키는 `Secure`, `HttpOnly`, `SameSite`를 적용한다.
- OAuth에는 세션에 묶인 일회용 `state`와 짧은 만료시간을 사용한다. 공급자가 PKCE를 지원하게 되면 추가 적용한다.
- 요청 크기 제한, 속도 제한, CSRF 방어, 감사 로그와 예산 제한을 적용한다.
- 개인정보 보존 기간과 사용자 삭제 기능을 제공한다.
- GitHub Pages 프런트 코드나 브라우저 저장소에는 비밀키를 넣지 않는다.
