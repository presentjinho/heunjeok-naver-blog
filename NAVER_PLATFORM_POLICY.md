# 네이버 플랫폼 연동 기준

확인일: 2026-07-11

## 네이버 로그인

- 인증 시작 주소는 `https://nid.naver.com/oauth2.0/authorize`로 고정합니다.
- `state`는 256비트 무작위 일회용 값이며 서버 세션에 묶고 5분 뒤 만료합니다. 원문 state는 저장하지 않고 SHA-256 키만 임시 보관합니다.
- 현재 공식 문서에서 PKCE 지원을 확인하지 못했으므로 지원되는 것처럼 구현하지 않습니다. 네이버가 공식 지원을 명시하면 별도 검증 후 추가합니다.
- 로그인 API에서 받은 회원 식별 해시 외 이름·이메일·전화번호·성별·연령·생일·프로필 사진은 요청하거나 저장하지 않는 것을 기본값으로 합니다.
- 액세스 토큰과 Client Secret은 서버 메모리/Secret 저장소 경계 밖으로 내보내지 않으며 브라우저 저장소·URL·감사 로그에 기록하지 않습니다.

## 블로그 연결의 의미

- 네이버 로그인 성공은 네이버 회원 인증이며, 사용자가 입력한 `blog.naver.com/{ID}`의 소유권을 자동으로 증명하지 않습니다.
- 소유권 확인 수단이 없는 동안 UI는 `네이버 로그인됨`과 `블로그 주소 등록됨`을 별도 상태로 표시합니다.
- 실제 소유 확인 전에는 자동 발행·댓글 등록·소유자 전용 데이터 접근 권한을 부여하지 않습니다.
- 네이버 블로그 글쓰기 Open API는 종료되었으므로 로그인 성공을 자동 발행 권한처럼 표현하지 않습니다.

## API 이용 제한

- 네이버 API 결과를 제3자에게 재판매하거나 네이버와 제휴한 서비스처럼 표시하지 않습니다.
- 검색 결과는 원문 의미를 변조하지 않고 출처·조회 시각·조회 범위를 표시합니다.
- 사용자 개인정보는 필요한 범위의 명시적 동의를 받기 전에는 수집·저장하지 않습니다.

## 검색광고 키워드 도구

- Search Ads API Key·Secret Key·Customer ID는 광고 계정 자격 증명이며 서버 Secret 저장소에서만 관리합니다.
- 서명은 `{timestamp}.GET./keywordstool` 문자열의 HMAC-SHA256 Base64 값으로 생성하고 Secret Key 자체는 요청 헤더에 보내지 않습니다.
- 키워드 도구가 제공하는 월간 PC·모바일 검색수와 경쟁 정도만 사용합니다. 이 응답은 CPC를 제공하지 않으므로 CPC를 추정하거나 블로그 수익 CPC로 표시하지 않습니다.
- `< 10` 값은 임의의 숫자로 치환하지 않고 그대로 표시합니다.
- 광고 계정 데이터의 공개 재제공 범위가 확인되기 전까지 `SEARCHAD_ENABLED=false`가 기본이며 사용자 UI에 노출하지 않습니다.
- 운영자가 네이버와의 계약·이용 범위·재제공 가능 여부를 확인한 뒤에만 기능 플래그를 활성화할 수 있습니다.
