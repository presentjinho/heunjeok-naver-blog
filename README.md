# 흔적 — 네이버 블로그 글쓰기 작업실

네이버 블로그용 경험 초안을 만들고 검수한 뒤 제목·본문·태그를 옮기기 쉽게 준비하는 모바일 우선 웹앱입니다. 네이버 공식 서비스가 아니며 로그인·발행·검색 노출을 대신하지 않습니다.

## 현재 기능

- 네이버 블로그 분야별 글감 템플릿
- 주제·말투·경험·훅·교정·실시간 탐색·발행 준비 기능별 바로가기
- 분야별 네이버 최신 블로그 검색 바로가기(조회수 순위로 표시하지 않음)
- 기존 글 30자 이상에서 말끝·문장 호흡·자주 쓰는 연결어를 추출하는 로컬 말투 참고 카드
- 사용자가 지정한 `쓰지 않는 표현`을 초안에서 다시 점검
- 방문 후기·제품 사용기·방법·비교·일상에 맞춘 핵심 3문항 + 선택 3문항 경험 인터뷰(완료 기준은 각 5자)
- 사용자가 답한 문장만 도입·과정·판단 문단으로 묶고, 문장 중간을 자르지 않는 제목·사진 순서·글 종류별 해시태그 생성
- `내 답변 / 문장 연결 / 관계 고지`를 구분하는 생성 당시 출처 기록과 `새로 만든 사실 0개` 표시
- 본문 수정 뒤 출처 미추적 상태 표시, 구버전 출처 패키지 복사 차단
- 상투 문구·과장·편집 표시·얇은 뼈대·긴 문단·반복 말끝·말투 카드 불일치·키워드 반복을 찾는 패턴 점검
- 내돈내산·제품 제공·원고료·제휴 링크 관계 구분과 첫머리 안내
- 입력 상태를 `추가 답변 필요 / 보완용 뼈대 / 답변 충분`으로만 표시하고 사실 검증처럼 표현하지 않음
- 제목·본문·해시태그 개별 복사와 전체 복사
- 5단계 위자드, 모바일 입력/결과/도구 탭, 첫 방문 데모
- 버전 비교, 해시태그 칩 편집, 협찬 고지 잠금, 모바일 미리보기
- 백업 v2 내보내기·가져오기(합치기/교체)와 저장 용량 안내
- 실제 발행 점검 체크박스
- 입력 변경 시 오래된 결과 복사 차단과 키보드·화면낭독기용 잠금 이유 안내
- 댓글 붙여넣기 → 편집 가능한 답변 틀 → 빈칸을 채운 뒤 복사; 원 댓글 변경 시 이전 후보 잠금
- 생성 전 경험 답변·메모·사진 설명의 자동 저장과 다음 방문 복구
- 말투 카드와 편집 초안의 브라우저 로컬 저장·전체 삭제

## 실행과 테스트

보안 헤더와 API 경계까지 확인하려면 파일을 직접 여는 대신 로컬 서버를 사용합니다.

```text
npm start
```

기본 주소는 `http://127.0.0.1:3000`입니다.

## 무료 공개판

`main` 브랜치가 갱신되면 GitHub Pages에 정적 공개판을 자동 배포합니다. 주제 정리, 말투 카드, 로컬 초안, 교정, 훅, 체크리스트와 기록은 모두 브라우저 안에서 작동합니다.

- 공개 주소: `https://presentjinho.github.io/heunjeok-naver-blog/`
- 호스팅 플랜: 공개 저장소의 GitHub Pages 무료 제공 범위
- 로그인·API 키·서버 비용: 없음

사진 취재함은 JPG·PNG·WebP 파일을 최대 10장까지 받습니다. 원본을 최대 1600px JPEG 사본으로 다시 만들어 EXIF·GPS 메타데이터를 제거하고 브라우저 IndexedDB에만 저장합니다. 사진은 서버·AI·JSON 백업 파일로 보내지지 않으며 사진 취재함의 별도 삭제 또는 전체 데이터 삭제로 지울 수 있습니다.

본문 편집 버전은 2초 동안 입력이 멈춘 시점의 서로 다른 본문만 최근 10개까지 로컬에 보관합니다. 버전 복구 전에 현재 본문을 먼저 저장해 복구 작업도 되돌릴 수 있습니다.

`index.html`을 브라우저에서 열면 실행됩니다. 빌드와 외부 API가 필요하지 않습니다.

```text
npm test
```

테스트는 Node.js 내장 테스트 러너와 jsdom·axe-core를 사용합니다. `npm ci` 후 실행하세요.

## 제품 경계

네이버는 외부 서비스용 블로그 글쓰기 API를 2020년 5월 종료했습니다. 따라서 자동 로그인, 자동 발행, 에디터 DOM 조작, 댓글 수집·자동 등록은 구현하지 않습니다. 제목·본문·태그를 일반 텍스트로 복사한 뒤 사용자가 네이버에서 확인하고 발행하는 흐름을 사용합니다.

- [네이버 블로그 글쓰기 API 종료 공지](https://developers.naver.com/notice/article/7527)
- [네이버 블로그 검색 노출 순서 기준](https://help.naver.com/service/5626/contents/22925?lang=ko)
- [네이버 검색광고 API](https://naver.github.io/searchad-apidoc/)
- [네이버 데이터랩 API](https://developers.naver.com/docs/serviceapi/datalab/search/search.md)

검색량·검색 추세·예상 입찰가 데이터는 아직 연결하지 않았습니다. 공식 API를 연결하기 전에는 가짜 숫자, 상위 노출 보장, 수익 예상액을 표시하지 않습니다.

현재 초안 생성은 외부 AI API가 아닌 로컬 규칙 기반입니다. 실제 AI 연결 전에도 없는 경험·가격·효과를 만들지 않도록 입력 근거를 구조화하는 데 초점을 맞췄습니다.

화면의 자동 문구 점검은 정해진 패턴 검사이며 AI 작성 여부, 사실 여부, 검색 품질을 판정하지 않습니다.

## 개인정보

- 붙여넣은 기존 글 원문은 저장하지 않습니다.
- 경험 답변·추가 메모·사진 설명, 분석한 말투 카드와 생성·편집한 초안은 현재 브라우저의 `localStorage`에 저장됩니다.
- 댓글 원문은 저장하지 않습니다.
- 화면의 `내 브라우저 데이터 삭제`로 말투 카드와 초안을 모두 삭제할 수 있습니다.
- 외부 서버로 보내는 데이터는 현재 없습니다.

## 실시간 탐색의 한계

네이버는 외부 서비스에 블로그 게시글 조회수나 분야별 조회수 순위를 제공하지 않습니다. 따라서 사이트는 “조회수 상위 3개”를 만들지 않고 분야별 네이버 최신 검색 링크만 제공합니다. 사용자가 참고 글을 직접 고른 뒤 원문을 저장하지 않는 구조 비교 기능을 사용할 수 있습니다.
