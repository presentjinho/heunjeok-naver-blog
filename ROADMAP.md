# 흔적 — 네이버 블로그 전용 로드맵

작성일: 2026-07-10  
시장 재조사: 2026-07-11

## 이번에 완료한 핵심 개선

- 5개 글 종류별 핵심 3문항 + 접힌 선택 3문항 경험 인터뷰.
- 답한 문장을 내부 검수 제목 없이 자연스러운 문단으로만 엮는 초안.
- `내 답변 / 문장 연결 / 관계 고지` 출처 표시와 새로 만든 사실 0개 계약.
- 상투 문구·과장·편집 표시·반복 말끝을 찾는 한계가 명시된 문장 패턴 점검.
- 자주 쓰는 연결어와 사용자가 지정한 금지 표현을 포함한 말투 카드.
- 입력하지 않은 경험·가격·효과를 생성하지 않는 로컬 생성 계약.
- 생성 전 경험 답변·메모·사진 설명 450ms 자동 저장과 다음 방문 복구.
- 입력 변경 뒤 이전 결과 복사 차단.
- 제목의 어절 중간 절단 제거와 글 종류별 해시태그 정리.
- 핵심·선택 질문 진행도 분리, 핵심 세 답변이 없을 때 정확한 입력으로 포커스 이동.
- 본문 편집 뒤 생성 당시 출처로 상태 전환, 구버전 출처 패키지 복사 차단.
- 미완성·빈 답변·원 댓글 변경 시 댓글 답변 복사 차단과 이유 안내.

다음 최우선은 모바일 사진·영수증 취재함과 최근 입력 버전 10개 복구다. 그다음 사용자가 고친 문장을 승인받아 말투 규칙으로 축적하고, 공식 Search Ads·DataLab API 자격과 약관을 확인한 뒤 실제 키워드 데이터 기능을 연결한다.

## 확정 범위

지원 대상은 네이버 블로그 하나다.

제공:

- 네이버 검색 데이터 기반 글감 탐색
- 개인 말투·경험 기반 초안
- 사진 순서·해시태그·광고 관계·발행 설정 점검
- 제목·본문·태그 일반 텍스트 복사
- 댓글 답변 후보 생성과 복사

제외:

- 네이버 외 플랫폼·SNS용 변환
- Markdown·HTML 발행 출력
- 네이버 자동 로그인·자동 발행·에디터 조작
- 댓글 자동 수집·등록
- 상위 노출·수익·실제 CPC 보장

## 1. 모바일·접근성 완성

- 320×568, 360×800, 390×844, 412×915, 768×1024, 가로 844×390 검증.
- 모든 폭에서 가로 스크롤 없음.
- 입력 16px 이상, 터치 대상 44×44px 이상.
- iOS Safari, Android Chrome, Samsung Internet, 네이버 앱 인앱브라우저 확인.
- 200%·400% 확대, 키보드 탐색, VoiceOver·TalkBack·NVDA 확인.
- WCAG AA 대비와 axe serious/critical 0건.
- 외부 글꼴 실패, 느린 네트워크, 가상 키보드, safe-area 대응.
- 입력 변경 후 오래된 결과 복사 성공 0건.

## 2. 공식 네이버 검색 데이터

### 데이터랩

- 검색 추세는 최고값 100 기준의 상대 지수로만 표시.
- `실시간 검색량`이나 절대 검색수로 표현하지 않음.
- 조회 기간·기기·성별·연령 필터와 조회 시각 표시.
- 공식 문서: [DataLab 검색어 트렌드](https://developers.naver.com/docs/serviceapi/datalab/search/search.md)

### 검색광고 키워드 도구

- 최근 30일 PC·모바일 검색수와 광고 경쟁 참고값을 서버에서 조회.
- 검색광고 계정의 API 라이선스와 Secret은 서버에만 보관.
- 검색수가 `<10`인 응답을 임의 숫자로 바꾸지 않음.
- 키워드 도구 값은 실시간 데이터가 아니라고 명시.
- 예상 입찰가는 블로그 수익이나 일반 키워드 실제 CPC로 표현하지 않음.
- 429 응답에 지수 backoff, 캐시, 호출 제한, 장애 안내 적용.
- 공식 문서: [Search AD API](https://naver.github.io/searchad-apidoc/)

### API 사용 검토 게이트

- API 결과 저장·캐시·가공·재배포 범위를 약관과 서면 문의로 확인.
- API 데이터 옆 광고나 API 결과 기반 유료 기능은 네이버 확인 전 보류.
- 모든 수치에 출처·기준 기간·조회 시각 표시.
- 장애 시 가짜 숫자 대신 `현재 데이터를 불러올 수 없음` 표시.

## 3. AI 초안 서버

- `POST /v1/tone/analyze`: 원문에서 편집 가능한 말투 프로필만 반환.
- `POST /v1/naver-drafts`: 주제·경험·말투·광고 관계로 구조화된 초안 반환.
- `POST /v1/comment-replies`: 사용자가 붙여넣은 댓글의 답변 후보만 반환.
- 원문 저장 기본값은 `아니오`; 처리 완료 후 폐기.
- 저장 선택 시 보존 기간, 내려받기, 삭제 제공.
- 입력 길이 제한, 출력 스키마 검증, HTML 정화, prompt injection 방어.
- API 키는 공개 저장소나 브라우저 빌드에 포함하지 않음.
- 생성 결과에는 검색 순위·수익 보장 문구를 허용하지 않음.
- 모든 문단에 `user_evidence_ids`, `connector_only`, `needs_current_check` 출처 상태 포함.
- 사용자가 제공하지 않은 1인칭 경험·가격·효과·통계 생성 금지.
- 사용자의 수정 내역은 자동 학습하지 않고 diff 미리보기와 승인 후 말투 규칙에 반영.

## 4. 네이버 에디터 인계

- `navigator.clipboard.writeText` 기반 일반 텍스트만 사용.
- 제목, 본문, 태그를 별도로 복사.
- 본문 복사에는 사진·장소·검수 안내를 섞지 않음.
- 사진 순서와 장소 첨부는 별도 체크로 안내.
- 네이버 블로그 새 탭 열기까지만 제공.
- 자동 로그인, 쿠키 접근, URL Scheme, DOM 주입, 반복 대량 발행 금지.
- 공식 글쓰기 API가 다시 제공될 때만 자동발행을 재검토.

공식 근거:

- [글쓰기 Open API 종료](https://developers.naver.com/notice/article/7527)
- [블로그 검색 제한 기준](https://help.naver.com/service/5593/contents/10586?lang=ko&osType=COMMONOS)
- [네이버 서비스 이용약관](https://policy.naver.com/rules/service.html)
- [네이버 콘텐츠 기본 가이드](https://searchadvisor.naver.com/guide/content-basic)

## 5. 공개 GitHub 웹앱

권장 구조:

```text
apps/
  web/                 # 모바일 웹앱
  api/                 # 검색 데이터·AI 서버 API
packages/
  contracts/           # 요청·응답 스키마
docs/
  privacy.md
  naver-platform-policy.md
  architecture.md
.github/workflows/
  ci.yml
  pages.yml
LICENSE
SECURITY.md
CONTRIBUTING.md
```

출시 순서:

1. 현재 로컬 기능과 모바일 검증 완료.
2. Git 저장소 초기화, 라이선스·보안·개인정보 문서 작성.
3. 타입·단위·모바일 E2E·접근성·빌드·secret scan CI 구성.
4. 비밀키 없는 프런트엔드를 GitHub Pages 공개 알파로 배포.
5. 검색 데이터·AI는 별도 서버 API에 배포하고 Secret은 서버에만 저장.
6. 커스텀 도메인 소유권 확인과 HTTPS 강제.

공개 차단 조건:

- 모바일·접근성 검사 통과.
- 오래된 결과 복사 차단과 로컬 데이터 삭제 검증.
- 경험 없는 입력에서 체험형 문장 생성 0건.
- 태그 30개 초과 0건.
- 자동발행·자동댓글·순위 보장 표현 0건.
- 저장소와 빌드 산출물의 API 키·토큰 노출 0건.
- 개인정보 안내와 네이버 비공식 서비스 표시 완료.

## 6. 운영 지표

- 주제 선택 → 초안 생성률.
- 초안 생성 → 제목·본문·태그 복사 완료율.
- `추가 답변 필요`에서 `보완용 뼈대`, 다시 `답변 충분`으로 개선된 비율.
- 생성기가 새로 만든 경험·수치 0건 유지율.
- 문장 패턴 점검에서 발견된 수정거리의 발행 전 해소율.
- 댓글 답변 후보 복사율.
- 7일 재방문율.
- 모바일 오류율과 복사 실패율.

검색 순위와 조회수는 통제할 수 없는 요소가 많으므로 제품 품질의 단독 성공지표로 사용하지 않는다.
