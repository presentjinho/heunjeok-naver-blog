보안 원칙

숨기는 미로보다, 애초에 덜 받는 구조.

보안은 복잡한 은닉이 아니라 공격할 데이터와 권한을 줄이는 데서 시작합니다. 공개판은 비밀번호·로그인 쿠키·API 키·자동 발행 권한을 받지 않습니다.

적용한 경계

  • 정적 GitHub Pages: 서버 계정과 결제형 API 없음
  • 일반 텍스트 복사만 제공: 네이버·X 에디터 조작 없음
  • CSP로 외부 스크립트·프레임·폼 전송 제한
  • 사용자 입력은 HTML로 실행하지 않고 텍스트로만 표시
  • 외부 링크는 새 창 권한 분리와 referrer 차단
  • 사진은 브라우저에서 축소·재인코딩하고 서버 전송 없음

사용자가 확인할 것

발행 전 가격·시간·주소·효과와 광고 관계를 직접 확인하세요. JSON 백업 파일에는 작성 데이터가 들어 있으므로 본인 기기에 보관하고 공개 저장소에 올리지 마세요.
